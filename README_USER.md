# ChenFlow Workbench 用户使用说明

这份文档面向实际使用 ChenFlow Workbench 的作者和读者，不讲代码结构，也不要求你了解 GraphRAG、Vue、FastAPI 这类开发细节。

如果你需要部署、改代码、排查环境问题，请看开发者文档 [README.md](./README.md)。

## 这是什么

ChenFlow Workbench 是一个本地优先的中文小说创作与阅读工作台。

你可以把它理解成两部分：

- `作者侧`：整理项目设定、人物卡、章节前提，生成草稿，发布正式作品
- `读者侧`：浏览公开作品、阅读章节、点赞、收藏、评论

它不是“点一下 AI 自动写完整本书”的工具。它更像一个把设定、检索、草稿、正式采用、发布管理串起来的写作工作流。

## 适合谁

- 正在写中文小说、轻小说、都市情感、悬疑、轻科幻的作者
- 想把“设定管理”和“正文创作”放在同一个工具里的人
- 想在本地环境里控制作品、草稿和资料的人

## 你能做什么

### 作者侧

- 注册账号并登录
- 创建小说项目
- 管理项目文件夹
- 维护项目设定
- 维护人物卡、长期记忆、参考资料
- 规划章节
- 生成章节草稿
- 从草稿发布正式作品
- 给已发布作品追加新章节
- 编辑已发布作品与章节

### 读者侧

- 浏览公开作品
- 打开作品详情
- 阅读章节
- 点赞
- 收藏
- 评论

## 开始前

如果你已经能打开下面这个地址，说明系统已经启动好了：

```text
http://127.0.0.1:8000
```

如果打不开，请让负责部署的人先按开发者文档完成启动。

## 第一次使用

### 1. 创建账号

打开首页后：

1. 点击 `创建账号`
2. 输入用户名和密码
3. 完成简单验证码
4. 注册并自动登录

如果你已经有账号，直接点 `登录`。

### 2. 创建你的第一个项目

登录后建议先做这几件事：

1. 点击 `创建项目`
2. 填写标题、题材、世界观、写作偏好
3. 进入 `人物卡`
4. 进入 `新增章节`
5. 为第一章写一个清晰的章节前提

## 推荐写作流程

推荐按下面的顺序使用，不要一上来就直接让系统生成正文。

### 第一步：立项目

先把这些内容写清楚：

- 作品标题
- 题材类型
- 世界观简介
- 写作偏好
- 主要人物卡

这一步越清楚，后面的草稿越稳定。

### 第二步：规划章节

每一章至少写出：

- 章节标题
- 这一章要发生什么
- 这一章推进哪条关系、冲突或信息

章节前提越具体，生成结果越可控。

### 第三步：准备检索资料

在正式生成草稿前，先把项目资料准备完整，然后执行索引。

只有项目状态准备完成后，系统才允许生成草稿。

## 草稿和正式章节的区别

这是使用时最重要的一条规则：

`草稿只是候选，只有采用或发布之后，才算正式内容。`

这意味着：

- 草稿可以反复比较、筛选、放弃
- 草稿不会自动成为作品正文
- 只有你明确发布或追加章节，才会进入正式作品

如果你把系统当作“草稿生成器”，它会更稳定；如果你把它当作“自动完稿器”，体验会差很多。

## 如何生成草稿

在项目已经准备好、索引完成后：

1. 进入某个章节
2. 输入这一章想怎么推进
3. 点击生成
4. 在草稿箱里查看候选结果

建议你比较关注这几项：

- 这一稿有没有偏题
- 人物语气是否稳定
- 关系推进是否合理
- 是否和项目设定冲突

## 如何把草稿变成正式作品

你有两种常用方式：

### 方式 1：发布为新作品

适合项目第一次对外发布。

### 方式 2：追加到已发布作品

适合你的作品已经上线，现在要继续更新章节。

注意：

- 发布后才算正式采用
- 采用新章节后，如果你想让后续生成吸收这些新内容，通常还需要重新索引

## 读者如何使用

如果你只是来读作品，可以直接走这条路径：

1. 首页看热门作品
2. 进入 `发现`
3. 搜索作品
4. 打开作品详情
5. 点击 `开始阅读`

登录后还可以：

- 点赞
- 收藏
- 评论

## 手机上怎么用

手机端现在使用顶部菜单：

- 点 `打开菜单` 展开导航
- 点 `收起菜单` 关闭导航

主要阅读和浏览已经适配手机宽度，但如果你要大量编辑项目设定或长文本，仍然更建议用桌面端。

## 常见问题

### 1. 为什么点了生成没反应

先检查：

- 你是否已经登录
- 项目是否已经创建
- 是否已经选中章节
- 章节前提是否写了
- 项目索引是否完成

### 2. 为什么草稿不能直接算正式章节

因为系统把“候选文本”和“正式正典”分开了。这样能避免错误草稿污染后续检索和生成。

### 3. 为什么发布新章节后，下一次生成还像没记住

因为正式内容通常要在重新索引后，才会稳定进入后续检索。

### 4. 为什么有些操作要求先登录

下面这些动作都需要登录：

- 进入我的小说
- 查看我的喜欢/收藏
- 查看回收站
- 点赞
- 收藏
- 评论

## 使用建议

- 先写清设定，再生成草稿
- 一章一章推进，不要一次塞太多目标
- 把草稿当候选版本，不要第一稿就直接发布
- 经常回头整理人物卡和长期设定
- 正式采用后，记得做下一轮索引
