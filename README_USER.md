# ChenFlow Workbench 使用说明

这份文档面向实际使用 ChenFlow Workbench 的作者，重点只讲怎么用当前工作台写小说。

## 你能用它做什么

- 创建一个小说项目
- 保存世界设定、写作规则、参考资料和人物卡
- 为每一章写章节前提
- 让系统生成草稿
- 查看生成过程和变化抽取结果
- 把草稿发布成作品，或追加到已发布作品里

## 推荐使用顺序

### 1. 新建项目

先把这些长期有效的信息写清楚：

- 标题
- 题材
- 世界设定
- 写作规则
- 参考作品

项目层放的是长期信息，不要把某一章的具体剧情写进这里。

### 2. 补人物卡和长期资料

建议至少补这两类内容：

- 主要人物是谁，现在是什么状态
- 长期不会轻易变化的规则、关系、背景

这些信息会参与后续草稿生成。

### 3. 新建章节

每个章节至少写清楚：

- 章节标题
- 这一章必须发生什么
- 这章的冲突、关系推进或结果

章节前提越具体，草稿越容易贴题。

### 4. 生成草稿

进入章节后输入你这次想写的内容，比如：

- 这一章要发生什么
- 哪两个角色必须碰面
- 哪个冲突必须升级
- 这一章要落到什么结果

系统会结合项目设定、人物卡、长期资料和章节前提生成草稿。

### 5. 查看生成过程

“生成过程”页主要用来查看：

- 当前阶段跑到哪里
- 使用了怎样的上下文摘要
- 草稿生成、润色、意图检查、变化抽取的摘要

它不是给你看底层 prompt 的地方。

### 6. 发布或追加章节

如果草稿可用：

- 可以直接发布成一部新作品的第一章
- 也可以把它追加到已发布作品里作为新章节

发布或追加后，系统会把这次采用的内容作为正式结果处理。

## 使用建议

- 项目设定写“长期真相”，章节前提写“本章必须发生的事”
- 不要把多个完全无关的要求塞进一次生成
- 如果生成不够贴题，优先改章节前提和本次提示词，不要一味堆长文本
- 人物卡和长期资料比空泛的世界观描述更能稳定生成效果

## 本地启动

如果你在本机运行这个项目，通常这样启动：

1. 启动 MySQL
2. 构建前端
3. 启动后端

也可以直接运行仓库里的：

- `start-workbench.bat`
- `scripts/start-workbench.ps1`
