# 晨流写作台 · ChenFlow Workbench

`晨流写作台（ChenFlow Workbench）` 是一个面向中文轻小说创作的本地工作台。

它的目标不是只帮你“生成一段字”，而是把下面这些事情串成一条连续创作链路：

- 管理项目设定
- 记录角色与世界的长期设定
- 补充参考资料
- 基于 GraphRAG 做上下文检索
- 生成章节正文
- 提取角色、关系、事件和外界看法的演化变化
- 将生成结果发布到书城并继续追加章节

如果你想做的是“能持续写下去的小说”，而不是一次性的随机片段，这个项目就是为这个方向设计的。

---

## 这个工具适合谁

适合：

- 想写中文轻小说或青春向小说的人
- 想让角色关系和世界设定长期连续的人
- 想把“设定、资料、生成、发布、章节续写”放在一个工作台里管理的人

不适合：

- 只想要一个超轻量的一次性文本生成器
- 不需要设定连续性和角色演化的人

---

## 你能做什么

### 1. 建立小说项目

你可以为每一部小说单独创建项目，填写：

- 项目标题
- 题材类型
- 故事前提
- 世界设定
- 写作规则
- 文风预设

### 2. 维护长期设定

“长期设定”是后续写作中必须长期保留的重要信息，例如：

- 角色核心性格
- 关系背景
- 世界规则
- 长期伏笔
- 写作硬约束

### 3. 补充参考资料

你可以持续加入参考资料，让后续生成更稳：

- 角色背景
- 地点资料
- 场景素材
- 设定补充
- 风格参考

### 4. 用 GraphRAG 组织上下文

系统会把你的项目资料、长期设定、参考资料和后续演化变化送进 GraphRAG。

生成正文前，系统会先检索：

- 当前最相关的局部信息
- 更大范围的全局信息
- 最近几章的角色状态和关系变化

然后整理成“写作上下文卡”再交给模型。

### 5. 生成正文

你输入本次写作目标后，系统会：

1. 检索上下文
2. 整理写作上下文卡
3. 生成初稿
4. 按需要做轻量润色

你还可以通过“功能调整”开关控制：

- 是否使用全局检索
- 是否使用写作上下文卡
- 是否启用轻量润色
- 是否写回演化状态

### 6. 让小说“演化”

每次生成正文后，系统会自动提取：

- 角色状态变化
- 关系变化
- 关键事件
- 外界认知变化

这些变化会被写回系统，并在下一章继续参与生成。

也就是说，角色不会永远停留在初始设定里，而是会随着章节真的发生变化。

### 7. 发布到书城

你可以把生成结果发布成书城作品，并设置：

- 标题
- 作者名
- 简介
- 宣传语
- 可见性

发布后还可以继续：

- 编辑作品信息
- 追加新章节
- 查看评论
- 管理书架和互动

## 快速开始

## 运行前准备

本机需要先安装：

- Python 3.11
- Node.js 18+
- Docker Desktop
- Ollama

并提前拉取本地 embedding 模型：

```powershell
ollama pull bge-m3
```

---

## 一键启动

推荐直接双击项目根目录里的 [start-workbench.bat](E:\Computer\Wyc_Xc\MVP\start-workbench.bat)。

也可以命令行运行：

```bat
cd /d E:\Computer\Wyc_Xc\MVP
start-workbench.bat
```

启动脚本会自动：

1. 检查 `Docker`、`Python`、`npm`、`Ollama`
2. 检查本地 `bge-m3`
3. 启动 `mysql` 和 `neo4j`
4. 如有需要执行前端依赖安装
5. 构建前端
6. 启动后端服务

启动成功后直接打开：

- 主页面：`http://127.0.0.1:8000`
- 健康检查：`http://127.0.0.1:8000/api/bootstrap`
- Neo4j Browser：`http://127.0.0.1:7474`

## 手动启动

如果你想自己分步调试：

### 1. 启动基础服务

```powershell
docker compose up -d mysql neo4j
```

### 2. 启动后端

```powershell
python -m app.api
```

### 3. 启动前端开发模式

```powershell
cd frontend
npm install
npm run dev
```

前端开发地址：

- `http://127.0.0.1:5173`

## 建议使用流程

1. 注册并登录
2. 创建小说项目
3. 填写故事前提、世界设定、写作规则
4. 补充长期设定和参考资料
5. 点击“整理资料并开始准备”
6. 等待项目进入可写作状态
7. 输入当前场景目标
8. 按需调整功能开关后生成正文
9. 查看写作上下文卡与演化快照
10. 将生成结果发布到书城
11. 在作品详情页继续编辑作品和追加章节

## 常见问题

### 1. 启动脚本提示没有 `bge-m3`

先执行：

```powershell
ollama pull bge-m3
```

### 2. 页面打不开

优先检查：

- Docker Desktop 是否已经启动
- `ollama` 是否正在运行
- `mysql` 和 `neo4j` 容器是否正常启动
- 当前窗口里后端服务是否还在运行

### 3. 想确认当前是不是走本地 Ollama embedding

访问：

```text
GET /api/bootstrap
```

返回里会包含：

- `embedding_model`
- `embedding_provider`
- `embedding_base_url`

### 4. 为什么生成会越来越慢

因为系统不是单次直接吐正文，而是可能同时启用了：

- 全局检索
- 写作上下文卡
- 轻量润色
- 演化状态写回

这些都能提升质量，但也会增加耗时。可以在“功能调整”里按需要开关。

## 当前重点

这个项目当前的重点不是“做一个壳子”，而是持续提升两件事：

1. 小说质量  
让生成结果更轻、更顺、更像好看的中文轻小说，而不是厚重散文化 prose。

2. 角色与关系演化  
让人物状态、关系、事件后果和外界认知可以被系统真正继承，而不是每章重新开始。

如果你想做的是“能持续长出来的小说”，这正是晨流写作台正在解决的方向。
